# Plan

这个目录用于维护“正在开发中的功能”的步骤计划，重点记录每个功能当前准备怎么做、按什么顺序推进、做到哪一步。

## 使用方式

- 一个正在开发的功能对应一个计划文件，文件名应尽量直接体现功能主题。
- 计划文件只记录当前开发周期内仍然有效的步骤拆解、阶段目标、依赖关系和阶段进展。
- `docs/in-progress.md` 负责维护“当前在做什么”以及对应计划文件入口；具体步骤不要重复写回 `docs/in-progress.md`。
- 功能稳定完成后，对应计划文件应停止继续膨胀；稳定结论迁移到 `docs/system-state.md`，如有必要再保留一个简短结果摘要或直接删除该计划文件。

## 推荐结构

- 背景：说明这个计划解决什么问题、当前边界是什么。
- 目标：列出本轮开发要达到的结果。
- 步骤：按顺序列出阶段任务、依赖和验收点。
- 进展：标记已完成、进行中、待开始的事项。
- 风险与待确认项：记录还没有定论、需要额外验证或可能影响方案的点。

## 命名建议

- 优先使用 `功能主题-plan.md` 这样的命名方式。
- 如果同一主题需要拆成多轮，可以在文件名里补充阶段，例如 `realtime-asr-phase1-plan.md`。
